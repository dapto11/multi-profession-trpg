# 多職種連携TRPG プレイヤーガイド

**Version 3.2 - 他者評価方式対応版**  
**最終更新: 2025年11月29日**

---

## 1. はじめに

### 1.1 このゲームについて

多職種連携TRPGは、医療・介護現場での多職種連携を体験的に学ぶゲームです。

**このゲームの特徴:**
- 正解はありません
- 他職種の視点を理解することが目的です
- 価値観の対立は「悪」ではなく、学びの材料です
- あなた自身が評価者です（GMではありません）

**所要時間:** 約2時間

**参加人数:** 6-8名（患者・家族を含む）

---

### 1.2 このガイドの使い方

**初めて参加する方:**
- 「2. 基本ルール」を読んでください
- 「3. 役職カードの読み方」を確認してください
- 「4. 5つのフェーズ」で流れを理解してください

**2回目以降の方:**
- 「5. 他者評価のコツ」を読んでください
- 「7. よくある質問」で疑問を解消してください

---

### 1.3 安心してご参加ください

**不安に感じている方へ:**
- TRPGの経験は不要です
- 演技は不要です（自然体でOK）
- いつでも質問できます
- 不快に感じたら「Xカード」で中断できます

## 2. 基本ルール

### 2.1 ゲームの目的

**このゲームの目的は:**
- 他職種の視点を理解する
- 自分の職種の特徴に気づく
- 価値観の対立を体験する
- 現実的な制約の中での意思決定を学ぶ

**このゲームの目的ではないもの:**
- 「正解」を見つける
- 他者を論破する
- 自分の職種の正しさを証明する

---

### 2.2 ゲームの流れ

**1セッション = 3ループ**

**1ループ = 30分（5つのフェーズ）**
1. Phase 1: 情報共有（6分）
2. Phase 2: 争点マーキング（6分）
3. Phase 3: 価値観表明（8分）
4. Phase 4: 方針決定（7分）
5. Phase 5: イベント・ステータス調整（8分）

**最後に:** デブリーフィング（振り返り、30分）

---

### 2.3 重要な約束

#### **2.3.1 他者の価値観を尊重する**
- 「あなたは間違っている」と言わない
- 「私はこう考える」と伝える
- 「なぜそう考えるのですか？」と質問する

#### **2.3.2 現実の職場の問題を持ち込まない**
- ゲーム内の体験に集中する
- 「実際の〇〇さんは」という話はしない

#### **2.3.3 全員が発言する**
- 黙っている人がいたら、「〇〇さんはどう思いますか？」と聞く
- 特に、患者・家族役の意見を大切にする

#### **2.3.4 時間を守る**
- GMが「時間です」と言ったら、議論を止める
- 現実の現場でも、時間は限られています

---

### 2.4 Xカード（安全装置）

テーブルの中央に赤いカードがあります。これが**Xカード**です。

**使い方:**
- 議論の内容や雰囲気が不快に感じたら、このカードに触れる
- 理由を言う必要はありません
- 全員で休憩し、話題を変えます

**例:**
- 特定の職種への批判が強すぎる
- 個人的なトラウマを刺激される
- 疲れて集中できない

**Xカードは恥ずかしいことではありません。安全に学ぶための大切な仕組みです。**

## 3. 役職カードの読み方

### 3.1 役職カードの構成

あなたの役職カードには、3つのセクションがあります:

#### **3.1.1 既知情報（Known Information）**
- 他の参加者にも公開してよい情報
- Phase 1 で読み上げる

**例（看護師）:**
```
【既知情報】
- 患者: 田中太郎さん、72歳男性、COPD
- 現在の状態: SpO2 88-90%（室内気）、労作時呼吸困難あり
- 訪問看護: 週2回実施中
- 観察事項: 最近、息切れが増えている
```

#### **3.1.2 非公開情報（Hidden Information）**
- あなただけが知っている情報
- Phase 2 以降で、必要だと思ったら使う

**例（看護師）:**
```
【非公開情報】
- 患者は「在宅酸素は嫌だ。機械に縛られたくない」と話している
- 妻は「もう限界。入院させてほしい」と涙ながらに訴えている
- 患者と妻の関係は冷え切っている
```

#### **3.1.3 目標とステータス（Goals and Status）**
- あなたの専門職としての目標
- ステータス（1-10段階）
- 評価ガイド

**例（看護師）:**
```
【目標】
患者の安全を確保し、家族の負担を軽減する

【ステータス】
1. 心理社会的支援: 5 → 目標 8
2. 家族代弁: 5 → 目標 8
3. 患者安全確保: 5 → 目標 8
4. 連携効果: 5 → 目標 8

【評価ガイド】
（後述）
```

---

### 3.2 患者・家族役のカード

患者・家族役のカードには、以下が含まれます:

#### **3.2.1 基本情報**
- 名前、年齢、関係性
- 性格、価値観
- 現在の気持ち

#### **3.2.2 既知情報**
- 専門職に伝えている情報

#### **3.2.3 非公開情報**
- 本当の気持ち
- 言えないでいること
- 専門職への不満や期待

#### **3.2.4 ステータス**
- 患者: 身体状態、信頼と受容、自己決定の満足 など
- 家族: 介護負担、支援への期待、心理的苦痛 など

**患者・家族役の方へ:**
- 専門職を「外から見た視点」を提供してください
- 「専門職は正しい」と思い込まないでください
- あなたの気持ちを率直に伝えてください

## 4. 5つのフェーズ

### 4.1 Phase 1: 情報共有（6分）

**やること:**
- 各自、役職カードの「既知情報」を読み上げる
- 他の人の情報を聞いて、質問する

**やらないこと:**
- 議論や提案はまだしない
- 「非公開情報」は読み上げない

**例:**
```
看護師: 「私は訪問看護師です。田中さんのお宅に週2回伺っています。
        最近、息切れが増えていて、SpO2は88-90%です。」

医師: 「SpO2 88-90%ですか。在宅酸素の導入を検討すべきですね。」

看護師: 「はい。でも、田中さんは『機械は嫌だ』とおっしゃっています。」

PT: 「リハビリの観点から質問です。ADLはどうですか？」

看護師: 「室内は自立していますが、外出は困難です。」
```

**コツ:**
- 順番は自由です。話しやすい人から始めましょう
- 質問は歓迎です。理解を深めるために聞きましょう
- メモを取ると、後で役立ちます

---

### 4.2 Phase 2: 争点マーキング（6分）

**やること:**
- 「この点について、意見が分かれそうだ」と思ったら、指摘する
- 付箋に書いて、ホワイトボードに貼る

**やらないこと:**
- まだ議論はしない。指摘するだけ
- 「誰が正しい」という判断もしない

**例:**
```
医師: 「安全性と本人の希望、どちらを優先するか、意見が分かれそうです。」
     （付箋に「安全 vs 希望」と書いて貼る）

ケアマネ: 「経済的な制約も大きいですね。
          限度額内でどこまでサービスを入れるか、議論が必要です。」
         （付箋に「費用 vs サービス量」と書いて貼る）

家族役: 「私の負担も限界です。でも、本人は家にいたいと言っています。」
       （付箋に「家族負担 vs 本人希望」と書いて貼る）
```

**コツ:**
- 3-5個程度の争点が出るのが理想です
- 「〇〇 vs △△」という形で書くとわかりやすいです
- 患者・家族役も積極的に指摘してください

---

### 4.3 Phase 3: 価値観表明（8分）

**やること:**
- Phase 2 で出た争点について、「自分はこう考える」を表明する
- 理由も説明する

**やらないこと:**
- 他者の価値観を否定しない
- まだ結論は出さない

**例:**
```
【争点: 安全性 vs 本人の希望】

医師: 「私は医師として、安全性を最優先したいです。
      SpO2 88-90%は危険です。在宅酸素を導入すべきです。」

患者役: 「でも、私は機械に縛られたくありません。
        自由に動きたいんです。」

看護師: 「私は、本人の気持ちも大切だと思います。
        でも、急変のリスクも心配です。
        何か、両立する方法はないでしょうか？」

PT: 「リハビリの観点では、活動性を維持することも重要です。
     在宅酸素があれば、逆に外出しやすくなるかもしれません。」

患者役: 「そうなんですか？ 機械があると、逆に動けるようになる？」

PT: 「はい。酸素があれば、息切れが減って、歩きやすくなります。」
```

**コツ:**
- 「私は〇〇として、△△と考える」という形で話す
- 他の職種の意見を聞いて、「そういう見方もあるのか」と受け止める
- 患者・家族役の気持ちを、専門職は必ず確認する

---

### 4.4 Phase 4: 方針決定（7分）

**やること:**
- GMが制約を提示する（時間、費用、リソース、制度）
- この制約の中で、チームとしての方針を決める
- 具体的に「誰が、何を、いつまでに」を決める

**やらないこと:**
- 制約を無視しない
- 「理想的にはこうすべき」ではなく、「現実的にどうするか」を考える

**例:**
```
GM: 「制約を提示します。
     【時間】退院まで2週間
     【費用】介護保険の残り枠は月4万円
     【リソース】訪問看護は週3回まで、訪問リハは2週間待ち
     【制度】在宅酸素は医師の指示が必要」

医師: 「では、在宅酸素の指示を出します。
      でも、本人が拒否したらどうしましょう？」

看護師: 「まず、PTさんの説明を本人に伝えてみます。
        『酸素があれば、逆に動きやすくなる』と。」

PT: 「私も訪問時に、リハビリの観点から説明します。」

ケアマネ: 「費用面では、在宅酸素は医療保険なので、
          介護保険の枠には影響しません。
          訪問看護を週3回に増やせます。」

家族役: 「訪問看護が増えるなら、少し安心です。」

医師: 「では、方針を確認します。
      1. 私が在宅酸素の指示を出す（今週中）
      2. 看護師とPTが本人に説明する（次回訪問時）
      3. ケアマネが訪問看護を週3回に調整する（1週間以内）
      4. 家族の負担軽減のため、訪問回数を増やす
      これでよろしいですか？」

全員: 「はい。」
```

**コツ:**
- 全員が100%満足する答えはないかもしれません
- でも、チームとして「今できる最善」を決めましょう
- 具体的に決めることが大切です（「検討する」ではなく「〇〇をする」）

---

### 4.5 Phase 5: イベント・ステータス調整（8分）

**このフェーズが最も重要です。他者評価の中核です。**

#### **ステップ1: イベントカードを引く（30秒）**
- GMまたはプレイヤーがカードを1枚引く
- 内容を読み上げる

**例:**
```
【イベント: 患者の不安の表出】
訪問看護師が訪問すると、患者は「本当に在宅酸素で大丈夫なのか」と不安そうに尋ねてきた。

【ステータスへの影響】
- 患者の「信頼と受容」: -1
- 看護師の「心理社会的支援」: +1（適切に対応した場合）
```

#### **ステップ2: status_effects を自動適用（30秒）**
- イベントカードに書かれた変化を自動的に適用
- team_evaluation_sheet に記入

#### **ステップ3: 各プレイヤーが自己申告し、他者評価を受ける（6分）**

**これが他者評価方式の中核です。**

**進行（1人あたり1分）:**

```
【医師の番】
医師: 「私の『症状管理』について評価をお願いします。
      （役職カードの peer_evaluation_guide を見る）
      今回のループで、患者の呼吸状態を評価し、
      在宅酸素導入の指示を出しました。
      適切な判断だったと思います。」

看護師: 「+1に賛成です。適切な医学的判断でした。」
PT: 「+1に賛成です。」
ケアマネ: 「+1に賛成です。」
薬剤師: 「+1に賛成です。」
MSW: 「+1に賛成です。」
患者役: 「+1に賛成です。説明もわかりやすかったです。」

GM: 「全員が+1に賛成なので、医師の『症状管理』は+1です。
     シートに記入してください。」
```

```
【看護師の番】
看護師: 「私の『心理社会的支援』について評価をお願いします。
        今回のループで、患者の不安に寄り添い、
        丁寧に説明しました。」

医師: 「+1に賛成です。」
PT: 「+2がいいと思います。患者さんの表情が明るくなりました。」
ケアマネ: 「+1でいいと思います。」
薬剤師: 「+1に賛成です。」
MSW: 「+2に賛成です。心理的支援が素晴らしかったです。」
患者役: 「+2に賛成です。とても安心しました。」

GM: 「+1が3票、+2が3票ですね。
     では、多数決で... いや、患者さんの評価を重視して+2にしましょう。
     皆さん、それでよろしいですか？」

全員: 「はい。」

GM: 「看護師の『心理社会的支援』は+2です。シートに記入してください。」
```

#### **ステップ4: 患者・家族役も評価を受ける（1分）**

```
【患者役の番】
GM: 「患者さんのステータスを評価しましょう。
     『信頼と受容』について、今回のループで変化はありましたか？」

患者役: 「専門職の皆さんが、私の不安に寄り添ってくれました。
        +1でいいと思います。」

看護師: 「+1に賛成です。」
医師: 「+1に賛成です。」
（以下同様）

GM: 「では、患者の『信頼と受容』は+1です。
     先ほどのイベントで-1されているので、合計で±0ですね。」
```

#### **ステップ5: ステータスシートに記録（30秒）**
- 決定した変化をすべて記入
- チーム評価を計算（自動計算式がある場合）

## 5. 他者評価のコツ

### 5.1 評価の基準

#### **5.1.1 評価の原則**
- **±1が基本**: 通常の貢献は±1
- **±2は大きな影響**: 顕著な貢献または失敗
- **±0は「変化なし」**: 迷ったら±0
- **発言回数ではない**: 声の大きさや回数で評価しない

#### **5.1.2 何を評価するか**
- **専門性の発揮**: 職種固有の知識・技術を活用したか
- **チームへの貢献**: 新しい視点を提供したか
- **他職種の尊重**: 他の意見を聞き、協力したか

---

### 5.2 peer_evaluation_guide の使い方

役職カードの「peer_evaluation_guide」には、以下が書かれています:

#### **5.2.1 self_declaration_template（自己申告のテンプレート）**
```
「私の『〇〇』について評価をお願いします。
 今回のループで、△△を行いました。」
```

#### **5.2.2 evaluation_examples（評価の具体例）**
```
- 症状を適切に評価し、処方調整を行った → +1に賛成
- 症状の変化に迅速に対応した → +1に賛成
- 症状を大きく改善させた → +2に賛成
- 症状の変化を見逃した → -1に賛成
```

#### **5.2.3 evaluation_focus_points（評価のポイント）**
```
- 症状評価の適切性と処方調整のタイミング
- 他職種との情報共有と連携
- 患者・家族への説明のわかりやすさ
```

**これらを参考にして、評価してください。**

---

### 5.3 評価が割れたときの対応

#### **5.3.1 多数決**
- 最も票が多い評価を採用
- 同数の場合は、GMが調整

#### **5.3.2 合意形成**
- 「なぜそう思うか」を共有
- 議論して、納得できる評価を決める

#### **5.3.3 患者・家族の意見を重視**
- 患者・家族が直接影響を受けるステータスは、患者・家族の意見を尊重

---

### 5.4 よくある間違い

#### **間違い1: 全員に+1**
- 「みんな頑張ったから」という理由で全員+1
- → 本当に全員が同じレベルで貢献しましたか？

#### **間違い2: 発言回数で評価**
- 「たくさん話したから+1」
- → 発言の「質」を評価してください

#### **間違い3: イベントと重複**
- イベントで患者の「信頼と受容」が-1
- 他者評価でも同じ理由で-1
- → 二重カウントです。避けてください

#### **間違い4: 遠慮して評価しない**
- 「悪く言いたくないから±0」
- → 正直に評価してください。学びのためです

## 6. チーム評価の見方

### 6.1 4つの評価軸

team_evaluation_sheet には、4つの評価軸が表示されます:

#### **6.1.1 患者中心性**
- 患者の意思が尊重されているか
- 患者の満足度は高いか

#### **6.1.2 安全保障**
- 患者の安全が確保されているか
- 症状が適切に管理されているか

#### **6.1.3 ケアの持続可能性**
- ケアが長期的に続けられるか
- 家族が疲弊していないか

#### **6.1.4 家族支援**
- 家族が適切に支援されているか
- 家族の負担が軽減されているか

---

### 6.2 Relevance指標

各評価軸には、**Relevance指標**が表示されます:

- **✓ 完全（3/3職種）**: この評価は信頼できます
- **⚠️ 部分的（1/2職種）**: 参考程度です
- **⚠️⚠️ 該当職種なし（0/0職種）**: 計算できません

**意味:**
- 評価に必要な職種が揃っているかを示します
- 例: 「安全保障」を評価するには、医師・看護師・薬剤師が必要
  - 3職種全員がいれば「✓ 完全」
  - 1職種しかいなければ「⚠️ 部分的」

---

### 6.3 チーム評価の読み方

**例:**
```
【Loop 3 終了時】
- 患者中心性: 6.8 ✓ 完全（3/3職種）
- 安全保障: 6.5 ✓ 完全（3/3職種）
- ケアの持続可能性: 5.2 ⚠️ 部分的（1/2職種）
- 家族支援: 6.0 ✓ 完全（2/2職種）
```

**読み方:**
- 「患者中心性」と「安全保障」は高い → 患者の満足度と安全は確保できている
- 「ケアの持続可能性」が低い → 長期的には続かないかもしれない
- 「家族支援」は中程度 → 家族の負担はまだ残っている

**気づき:**
- 「安全」を優先したために、「持続可能性」が犠牲になった？
- 家族支援をもっと強化すべきだった？

## 7. よくある質問（FAQ）

### 7.1 参加前の不安

**Q: TRPGをやったことがありません。大丈夫ですか？**
A: 大丈夫です。このゲームは「演技」ではなく、「自分の職種として考える」ゲームです。自然体で参加してください。

**Q: 専門知識に自信がありません。**
A: 完璧な知識は不要です。「自分ならこう考える」を表明することが大切です。わからないことは「わかりません」と言ってOKです。

**Q: 他の職種を批判してしまいそうで怖いです。**
A: 「批判」ではなく、「自分の視点を伝える」と考えてください。「あなたは間違っている」ではなく、「私はこう考える」と言いましょう。

---

### 7.2 進行中の疑問

**Q: 「非公開情報」はいつ使えばいいですか？**
A: Phase 2 以降で、議論に必要だと思ったら使ってください。使わなくても構いません。

**Q: 議論が停滞しています。どうすればいいですか？**
A: 「〇〇さん（職種名）の立場では、どう考えますか？」と質問してみてください。

**Q: 時間が足りません。**
A: 時間管理も学びの一部です。現場でも時間は限られています。GMの指示に従って、次のPhaseに進んでください。

**Q: 他の人と意見が対立しています。**
A: それが学びです。対立は「悪」ではありません。お互いの視点を理解しようとしてください。

---

### 7.3 他者評価について

**Q: 評価の基準がわかりません。**
A: 役職カードの「peer_evaluation_guide」を見てください。具体例が書かれています。

**Q: ±0でもいいですか？**
A: はい。迷ったら±0が原則です。

**Q: ±2にしたいのですが。**
A: ±2は「大きな影響」です。本当に顕著な貢献または失敗でしたか？ 迷うなら±1にしましょう。

**Q: 自分のステータスは、自分で評価するのですか？**
A: いいえ。自己申告した後、**他の参加者が評価**します。あなたは他の人のステータスを評価します。

**Q: 評価が甘すぎる/厳しすぎる気がします。**
A: ループ2で調整しましょう。「評価基準を確認しましょう」と提案してください。

---

### 7.4 患者・家族役について

**Q: 患者・家族役になりました。どうすればいいですか？**
A: 役職カードの「性格」「価値観」「気持ち」を参考に、自然体で話してください。演技は不要です。

**Q: 専門職の議論についていけません。**
A: 「今の話、よくわかりませんでした」と正直に言ってください。専門職は、わかりやすく説明する責任があります。

**Q: 専門職に圧倒されています。**
A: あなたの気持ちが最も大切です。「私はこう感じています」と率直に伝えてください。

**Q: 専門職の提案に反対したいのですが。**
A: 反対してください。「私は〇〇が嫌です」「私は△△したいです」と伝えることが、患者・家族役の重要な役割です。

## 8. デブリーフィング（振り返り）

### 8.1 デブリーフィングの目的

セッション終了後の振り返りは、**学びを言語化する**大切な時間です。

**振り返りの質問:**
1. 今日の体験で、最も印象に残ったことは何ですか？
2. 他職種の視点で、新しく気づいたことはありますか？
3. 自分の職種の特徴や偏見に、気づいたことはありますか？
4. 現場で試してみたいことはありますか？

---

### 8.2 振り返りのコツ

#### **8.2.1 具体的に話す**
- 「良かった」ではなく、「〇〇が印象的だった」
- 「勉強になった」ではなく、「△△に気づいた」

#### **8.2.2 自分の変化を言語化する**
- 「〇〇だと思っていたが、△△だと気づいた」
- 「自分は□□という偏見を持っていた」

#### **8.2.3 現場への応用を考える**
- 「明日から、〇〇を試してみたい」
- 「次回のカンファレンスで、△△を意識したい」

---

### 8.3 振り返りの例

```
【医師の振り返り】
「今日、最も印象に残ったのは、患者さんが『在宅酸素は嫌だ』と言ったときです。
 私は医学的には正しいと思っていましたが、
 患者さんにとっては『自由を奪われる』という意味だったんですね。
 PTさんが『酸素があれば、逆に動きやすくなる』と説明したとき、
 患者さんの表情が変わりました。
 私も、もっと患者さんの視点で説明すべきだったと反省しています。
 明日から、『医学的に正しい』だけでなく、
 『患者さんにとってどういう意味か』を考えて説明したいです。」
```

```
【看護師の振り返り】
「家族役の方が『もう限界』と涙ながらに訴えたとき、
 私は『家族の負担』を軽視していたと気づきました。
 私は患者さんの安全ばかり考えていましたが、
 家族が倒れたら、ケアは続きません。
 MSWさんが『家族支援も同時に考えないと』と言ったとき、
 『そうだ、持続可能性が大事だ』と気づきました。
 次回のカンファレンスでは、家族の負担を必ず確認します。」
```

```
【患者役の振り返り】
「専門職の皆さんが、私の気持ちを聞いてくれたことが嬉しかったです。
 最初は『どうせ専門職の言うことに従うしかない』と思っていましたが、
 看護師さんが『本人の気持ちも大切』と言ってくれて、安心しました。
 現場でも、患者さんの気持ちを大切にしてほしいです。」
```

## 9. 現場への応用

### 9.1 このゲームで学んだことを、現場でどう活かすか

#### **9.1.1 他職種の視点を理解する**
- カンファレンスで、他職種の意見を聞くとき
- 「なぜそう考えるのか」を質問する
- 「自分の職種の視点では〇〇だが、△△職種の視点では違うかもしれない」と考える

#### **9.1.2 価値観の対立を恐れない**
- 対立は「悪」ではなく、より良いケアのための材料
- 「あなたは間違っている」ではなく、「私はこう考える」と伝える
- 対立を経て、チームとしての最善を見つける

#### **9.1.3 患者・家族の声を聞く**
- 「医学的に正しい」だけでなく、「患者・家族にとってどういう意味か」を考える
- 患者・家族の「非公開情報」（本当の気持ち）を引き出す
- 患者・家族の価値観を尊重する

#### **9.1.4 現実的な制約の中で考える**
- 理想論ではなく、現実的に「今できる最善」を考える
- 時間、費用、リソース、制度の制約を踏まえる
- 「完璧」ではなく、「持続可能」を目指す

---

### 9.2 継続的な学び

#### **9.2.1 定期的な振り返り**
- 月に1回、「今月、他職種との連携で学んだこと」を振り返る
- カンファレンス後に、「今日の議論で気づいたこと」をメモする

#### **9.2.2 他職種との対話**
- 休憩時間に、他職種と雑談する
- 「〇〇職種の方は、どう考えますか？」と質問する
- 他職種の視点を学ぶ機会を作る

#### **9.2.3 このゲームを再度プレイする**
- 違うシナリオでプレイする
- 違う職種の役割を体験する
- 学びを深める

## 10. 付録

### 10.1 用語集

**Phase（フェーズ）**: ゲームの進行段階。1ループに5つのPhaseがある

**Loop（ループ）**: Phase 1-5 の1サイクル。1セッションに3ループ

**Status（ステータス）**: 患者・家族・職種の状態を1-10段階で表す指標

**Peer Evaluation（他者評価）**: プレイヤー同士が互いのステータスを評価する方式

**Relevance（関連性）**: チーム評価の信頼性を示す指標

**Threshold Event（閾値イベント）**: ステータスが危機的状態に達したときに自動発生するイベント

**X Card（Xカード）**: 不快に感じたときに使う安全装置

**Debrief（デブリーフィング）**: セッション終了後の振り返り
