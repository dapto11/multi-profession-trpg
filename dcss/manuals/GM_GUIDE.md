# GM_GUIDE.md

# 多職種連携TRPG ゲームマスターガイド

**Version 3.2 - 他者評価方式対応版**  
**最終更新: 2025年11月29日**

---

## 1. はじめに

### 1.1 このガイドの目的

このガイドは、多職種連携TRPGのゲームマスター（GM）として、セッションを円滑に進行するための実践的な情報を提供します。

**対象読者:**
- 初めてGMを担当する方
- 複数グループを同時に見る方
- 他者評価方式に不慣れな方

---

### 1.2 GMの役割

多職種連携TRPGにおけるGMの役割は、**従来のTRPGのGMとは大きく異なります**。

#### **従来のTRPG GMの役割:**
- ストーリーを語る
- NPCを演じる
- ルール判定を行う
- プレイヤーの行動に反応する

#### **多職種連携TRPG GMの役割:**
- **進行管理**: 時間管理、フェーズ移行の合図
- **ルール説明**: 初回のみ、簡潔に
- **質問対応**: プレイヤーからの質問に答える
- **記録支援**: ステータスシートの記入を手伝う（必要に応じて）
- **観察**: プレイヤーの様子を観察し、介入の必要性を判断

#### **GMが行わないこと:**
- ステータス調整の判断（プレイヤーに委ねる）
- 争点の評価・判定（プレイヤーに委ねる）
- 「正解」の提示（多様な答えを尊重する）

---

### 1.3 複数グループの運営

本システムは、**1名のGMが2-4グループを同時に見る**ことを想定しています。

**可能な理由:**
- プレイヤー主導の進行
- 他者評価方式（GMの判断が不要）
- 明確なフェーズ構造

**GMの動き:**
- 各グループを巡回
- フェーズ移行のタイミングで全体に合図
- 質問があるグループに対応
- 記録の確認

### 2.3 役職カードの配布方法

#### **2.3.1 基本原則**
- 参加者の実際の職種と**同じ役職カード**を配布（推奨）
- 学習効果を高めるため、「自分の職種を深く理解する」ことを優先

#### **2.3.2 患者・家族役の配布**
- 患者役: 共感性が高く、演技に抵抗がない人
- 家族役: 介護経験がある、または家族支援に関心がある人

#### **2.3.3 配布時の説明**
```
「これから、皆さんには役職カードをお配りします。
カードには3つのセクションがあります:

1. 既知情報: 他の参加者にも公開してよい情報
2. 非公開情報: あなただけが知っている情報
3. 目標とステータス: あなたの専門職としての目標

カードをよく読んで、自分の役割を理解してください。
質問があれば、Phase 1 が始まる前にお答えします。」
```

## 3. フェーズ別進行ガイド

### 3.1 オープニング（5分）

#### **3.1.1 全体説明**

```
「本日は多職種連携TRPGにご参加いただき、ありがとうございます。
このゲームは、多職種連携における価値観の対立を体験し、
理解を深めることを目的としています。

【重要なルール】
1. 正解はありません。多様な意見を尊重してください。
2. 他者の価値観を否定しないでください。
3. 現実の職場での出来事は持ち込まないでください。
4. 不快に感じたら、いつでも「Xカード」を使えます。

【進行】
- 1ループは30分、全部で3ループ行います。
- 各ループは5つのフェーズに分かれています。
- 私はタイムキーパーとして時間をお知らせします。
- ステータスの調整は、皆さん自身で行います。

それでは、始めましょう。」
```

#### **3.1.2 Xカードの説明**

```
「テーブルの中央に赤いカードがあります。これが『Xカード』です。
もし、議論の内容や雰囲気が不快に感じたら、
このカードに触れてください。理由を言う必要はありません。
全員で一旦休憩し、話題を変えます。」
```

---

### 3.2 Phase 1: 情報共有（6分）

#### **3.2.1 開始時の説明**

```
「Phase 1: 情報共有を始めます。時間は6分です。

【やること】
- 各自、役職カードの『既知情報』を読み上げてください。
- 順番は自由です。
- 他の人の情報を聞いて、質問してもOKです。

【やらないこと】
- 議論や提案はまだしません。
- 『非公開情報』は読み上げないでください。

それでは、スタート！」
```

#### **3.2.2 GM の観察ポイント**

- [ ] 全員が発言しているか
- [ ] 患者・家族役が萎縮していないか
- [ ] 時間内に全員が情報共有できているか

#### **3.2.3 よくある質問と回答**

**Q: 「非公開情報はいつ使うんですか？」**
A: 「Phase 2 以降で、必要だと思ったら使ってください。使わなくても構いません。」

**Q: 「この情報は正しいんですか？」**
A: 「カードに書かれている情報は、その役職の視点では正しいです。ただし、他の職種から見ると違う解釈もあるかもしれません。」

---

### 3.3 Phase 2: 争点マーキング（6分）

#### **3.3.1 開始時の説明**

```
「Phase 2: 争点マーキングを始めます。時間は6分です。

【やること】
- 『この点について、意見が分かれそうだ』と思ったら、指摘してください。
- 例: 『安全性と本人の希望、どちらを優先するか』
- 付箋に書いて、ホワイトボードに貼ってください。

【やらないこと】
- まだ議論はしません。指摘するだけです。
- 『誰が正しい』という判断もしません。

それでは、スタート！」
```

#### **3.3.2 GM の観察ポイント**

- [ ] 争点が3-5個程度出ているか（多すぎても少なすぎてもNG）
- [ ] 8つの争点タグ（safety, time, resources等）がバランスよく出ているか
- [ ] 特定の職種だけが発言していないか

#### **3.3.3 介入が必要な場合**

**争点が出ない場合:**
```
「少しヒントを出しますね。
例えば、『時間的な制約』について、意見が分かれそうなポイントはありませんか？」
```

**争点が多すぎる場合:**
```
「たくさん出ましたね。この中で、特に重要だと思う3つに絞りましょう。」
```

### 3.4 Phase 3: 価値観表明（8分）

#### **3.4.1 開始時の説明**

```
「Phase 3: 価値観表明を始めます。時間は8分です。

【やること】
- Phase 2 で出た争点について、『自分はこう考える』を表明してください。
- 例: 『私は看護師として、安全性を最優先したい』
- 理由も説明してください。

【やらないこと】
- 他者の価値観を否定しないでください。
- まだ結論は出しません。

それでは、スタート！」
```

#### **3.4.2 GM の観察ポイント**

- [ ] 全員が自分の価値観を表明しているか
- [ ] 患者・家族役の価値観も尊重されているか
- [ ] 対立が建設的に進んでいるか（感情的になっていないか）

#### **3.4.3 介入が必要な場合**

**特定の職種が黙っている場合:**
```
「〇〇さん（職種名）の立場では、どう考えますか？」
```

**議論が感情的になっている場合:**
```
「一旦、落ち着きましょう。
今は『誰が正しいか』ではなく、『それぞれがどう考えているか』を共有する時間です。」
```

### 3.5 Phase 4: 方針決定（7分）

#### **3.5.1 開始時の説明**

```
「Phase 4: 方針決定を始めます。時間は7分です。

【制約の提示】
（制約セットカードを表向きにする）
『今回の制約は、こちらです。』
（制約を読み上げる）

【やること】
- この制約の中で、チームとしての方針を決めてください。
- 具体的に『誰が、何を、いつまでに』を決めましょう。

【重要】
- 全員が100%満足する答えはないかもしれません。
- でも、チームとして『今できる最善』を決めてください。

それでは、スタート！」
```

#### **3.5.2 制約の読み上げ例**

```
【時間制約】
- 退院まで2週間

【経済制約】
- 介護保険の限度額: 月27万円
- 現在の利用額: 月23万円（残り4万円）

【リソース制約】
- 訪問看護: 週3回まで（現在週2回）
- 訪問リハビリ: 2週間待ち

【制度制約】
- 在宅酸素療法: 医師の指示が必要
- 介護保険の区分変更: 申請から1ヶ月
```

#### **3.5.3 GM の観察ポイント**

- [ ] 制約を踏まえた議論ができているか
- [ ] 具体的な方針が決まっているか（「検討する」ではなく「〇〇をする」）
- [ ] 全員が納得しているか（少なくとも、理解はしているか）

#### **3.5.4 よくある質問と回答**

**Q: 「この制約は絶対ですか？」**
A: 「基本的には絶対です。ただし、緊急時には例外もあります。今回は緊急時ですか？」

**Q: 「もっと時間がほしい」**
A: 「現実の現場でも、時間は限られています。その中で決めることも学びの一部です。」

### 3.6 Phase 5: イベント・ステータス調整（8分）

#### **3.6.1 開始時の説明**

```
「Phase 5: イベント・ステータス調整を始めます。時間は8分です。

【進行】
1. イベントカードを1枚引きます（30秒）
2. イベントの影響を自動適用します（30秒）
3. 各自のステータスについて、他者評価を受けます（6分）
4. 患者・家族のステータスも評価します（1分）
5. ステータスシートに記録します（30秒）

それでは、イベントカードを引いてください。」
```

#### **3.6.2 イベントカードの読み上げ**

**GMが行う場合:**
```
（カードを引く）
「イベントが発生しました。」
（タイトルを読む）
「『〇〇〇〇』」
（本文を読む）
「〇〇〇〇〇〇〇〇〇〇」
（status_effects を確認）
「このイベントにより、以下のステータスが変化します:
- 患者の『信頼と受容』が -1
- 医師の『症状管理』が +1」
```

**プレイヤーが行う場合:**
```
「誰か、カードを引いて読み上げてください。」
```

#### **3.6.3 他者評価の進行（最重要）**

**3.6.3.1 基本的な流れ（1人あたり1分）**

```
【医師の番】
医師: 「私の『症状管理』について評価をお願いします。
      今回のループで、患者の呼吸状態を評価し、
      処方調整を提案しました。」

看護師: 「+1に賛成です。適切な評価でした。」
PT: 「+1に賛成です。」
ケアマネ: 「+1に賛成です。」
薬剤師: 「+1に賛成です。」
MSW: 「+1に賛成です。」

GM: 「全員が+1に賛成なので、医師の『症状管理』は+1です。
     シートに記録してください。」

【次の職種へ】
```

**3.6.3.2 評価が割れた場合**

```
【看護師の番】
看護師: 「私の『家族代弁』について評価をお願いします。
       家族の負担を観察し、チームに報告しました。」

医師: 「+1に賛成です。」
PT: 「+1に賛成です。」
ケアマネ: 「+2がいいと思います。家族の限界を見極めていました。」
薬剤師: 「+1でいいと思います。」
MSW: 「+2に賛成です。」

GM: 「意見が分かれましたね。
     +1が3票、+2が2票なので、多数決で+1です。
     シートに記録してください。」
```

**3.6.3.3 患者・家族のステータス評価**

```
GM: 「次に、患者さんのステータスを評価しましょう。
     『信頼と受容』について、今回のループで変化はありましたか？」

患者役: 「専門職の皆さんが、私の話をよく聞いてくれました。
        +1でいいと思います。」

看護師: 「+1に賛成です。」
医師: 「+1に賛成です。」
（以下同様）

GM: 「では、患者の『信頼と受容』は+1です。
     先ほどのイベントで-1されているので、
     合計で±0ですね。シートに記録してください。」
```

#### **3.6.4 GM の観察ポイント**

- [ ] 時間内に全員の評価が終わっているか
- [ ] 評価の根拠が明確か（「なんとなく」ではなく）
- [ ] 評価が公平か（声の大きい人の意見に流されていないか）
- [ ] 患者・家族のステータスも評価されているか

#### **3.6.5 よくある質問と回答**

**Q: 「±0でもいいですか？」**
A: 「はい、迷ったら±0が原則です。」

**Q: 「±2にしたいんですが」**
A: 「±2は大きな影響です。本当に顕著な貢献または失敗でしたか？
    もし迷うなら、±1にしておきましょう。」

**Q: 「イベントの影響と重複していませんか？」**
A: 「良い指摘ですね。同じ理由で二重に加点しないように注意してください。
    イベントは患者・家族の体験、他者評価は職種の貢献を評価します。」

```
「Loop 1 が終了しました。お疲れさまでした。

【現在のチーム評価】
（team_evaluation_sheet を確認）
- 患者中心性: 6.2

- 家族支援: 6.0

【次のループへ】
Loop 2 では、先ほど決めた方針を実行に移します。
新しいイベントが発生するかもしれません。
準備ができたら、Phase 1 から始めましょう。」
```

### 3.8 閾値イベントの処理

#### **3.8.1 閾値イベントとは**

患者・家族のステータスが危機的状態（通常は2以下、または8以上）に達すると、**自動的に発生する特別なイベント**です。

**例:**
- 患者の『信頼と受容』が2以下 → 「患者の治療拒否」
- 家族の『介護負担』が8以上 → 「家族の限界」

#### **3.8.2 発生時の処理**

```
（ステータスシートを確認）
GM: 「患者の『信頼と受容』が2になりました。
     閾値イベントが発生します。」

（閾値イベントカードを引く）
GM: 「『患者の治療拒否』」
（本文を読む）

「このイベントにより、今後5分間、
 『どうやって信頼関係を回復するか』を議論してください。
 他の議題は一旦保留です。」

【5分後】
GM: 「時間です。信頼関係を回復するための方針は決まりましたか？」

（方針が決まったら）
GM: 「では、患者の『信頼と受容』を+2してください。
     これで4になり、回復条件を満たしました。」
```

#### **3.8.3 注意点**

- 閾値イベントは**強制発生**（プレイヤーの選択ではない）
- 回復条件を満たすまで、他の議題に進めない
- 回復後は、冷却期間（通常1ループ）があり、同じイベントは再発しない

## 4. 記録と測定

### 4.1 team_evaluation_sheet の記入

#### **4.1.1 基本的な流れ**

1. **初期値の記入**（セッション開始前）
   - scenario_template.json の初期値をシートに転記

2. **ループごとの記入**（Phase 5 終了後）
   - イベントの status_effects を適用
   - 他者評価の結果を適用
   - 合計値を計算

3. **チーム評価の計算**（各ループ終了後）
   - 計算式に従って4つの評価軸を計算
   - Relevance指標を確認

#### **4.1.2 記入例**

```
【Loop 1 終了後】

ステータス          | 初期値 | Loop 1 | 変化
--------------------|--------|--------|------
患者: 身体状態       | 4      | 4      | 0
患者: 信頼と受容     | 6      | 5      | -1
家族: 介護負担       | 6      | 7      | +1
医師: 症状管理       | 5      | 6      | +1
医師: 連携効果       | 5      | 5      | 0
看護師: 家族代弁     | 5      | 6      | +1
看護師: 連携効果     | 5      | 6      | +1
...

【チーム評価】
患者中心性 = (5 + 6) / 2 × 0.6 + (5 + 6 + ...) / 職種数 × 0.4
          = 3.3 + 2.9 = 6.2

（以下同様に計算）
```

### 4.2 measurement_log.json の記録

#### **4.2.1 記録のタイミング**

- **セッション開始時**: 参加者情報、シナリオID、開始時刻
- **各ループ終了時**: status_snapshot, status_changes, threshold_events_occurred
- **セッション終了時**: session_metrics, qualitative_feedback

#### **4.2.2 重要な記録項目**

**status_changes の記録:**
```json
{
  "timestamp": "2025-11-29T19:50:00Z",
  "target": "profession.physician.symptom_management",
  "change": +1,
  "before": 5,
  "after": 6,
  "trigger": "peer_evaluation",
  "evaluation_method": "majority",
  "evaluators": [
    {"player_id": "P1", "role": "nurse", "vote": "agree"},
    {"player_id": "P2", "role": "pt", "vote": "agree"},
    {"player_id": "P3", "role": "care_manager", "vote": "neutral"}
  ],
  "self_declaration": "症状を適切に評価し、処方調整を行いました",
  "reason": "適切な処方調整を行った",
  "operator_role": "physician",
  "operator_player_id": "P0"
}
```

---

### 4.3 観察記録

GMは、以下の点を観察し、セッション後に記録することを推奨します:

#### **4.3.1 プロセスの質**
- [ ] 全員が発言できていたか
- [ ] 対立は建設的だったか
- [ ] 患者・家族役が尊重されていたか
- [ ] 時間管理は適切だったか

#### **4.3.2 学習の兆候**
- [ ] 「そういう見方もあるのか」という発言
- [ ] 「自分の職種では〇〇と考えがちだが」という自己省察
- [ ] 他職種への質問や確認
- [ ] 価値観の変化の兆し

#### **4.3.3 介入が必要だった場面**
- どのような場面で介入したか
- 介入の効果はあったか
- 次回改善すべき点

## 5. トラブルシューティング

### 5.1 時間管理の問題

#### **問題: Phase が時間内に終わらない**

**原因:**
- 議論が白熱している
- 争点が多すぎる
- 特定の職種が長く話している

**対応:**
```
「時間が来ました。この議論は重要ですが、
 一旦ここで区切りましょう。
 次のPhaseでも続けられます。」
```

**予防策:**
- 残り1分で「あと1分です」と予告
- タイマーを見えるところに置く
- ループ2以降は効率化を促す

---

#### **問題: Phase が早く終わってしまう**

**原因:**
- 争点が少ない
- プレイヤーが消極的
- 議論が浅い

**対応:**
```
「まだ時間がありますね。
 他に気になる点はありませんか？
 例えば、〇〇についてはどう考えますか？」
```

---

### 5.2 参加者の問題

#### **問題: 特定の職種が黙っている**

**原因:**
- 自信がない
- 他の職種に圧倒されている
- 役割が不明確

**対応:**
```
「〇〇さん（職種名）の立場では、どう考えますか？
 専門職として、気になる点はありませんか？」
```

**予防策:**
- Phase 1 で全員が発言する雰囲気を作る
- 患者・家族役を積極的に巻き込む

---

#### **問題: 議論が感情的になっている**

**原因:**
- 価値観の対立が激しい
- 現実の職場の問題が持ち込まれている
- 疲労

**対応:**
```
「一旦、落ち着きましょう。
 今は『誰が正しいか』を決める場ではありません。
 『それぞれがどう考えているか』を理解する場です。
 5分間、休憩しましょう。」
```

---

#### **問題: 患者・家族役が萎縮している**

**原因:**
- 専門職に圧倒されている
- 役割が不明確
- 発言のタイミングがわからない

**対応:**
```
「患者さん（または家族さん）は、今の話をどう感じましたか？
 専門職の皆さんは、患者さんの気持ちを聞いてください。」
```

**予防策:**
- Phase 1 で患者・家族役を最初に発言させる
- 「患者さんの気持ちを確認しましょう」と定期的に促す

---

### 5.3 ステータス調整の問題

#### **問題: 評価が甘すぎる（全員+1）**

**原因:**
- 評価基準が不明確
- 遠慮している
- 「みんな頑張った」という雰囲気

**対応:**
```
「±1は『基本的な貢献』です。
 本当に全員が同じレベルで貢献しましたか？
 もう一度、peer_evaluation_guide を確認してください。」
```

---

#### **問題: 評価が厳しすぎる（全員-1）**

**原因:**
- 完璧主義
- 自己批判的
- ネガティブな雰囲気

**対応:**
```
「±0が基本です。迷ったら変化させないでください。
 -1は『明らかな問題があった』場合のみです。」
```

---

#### **問題: 評価の根拠が不明確**

**原因:**
- 「なんとなく」で評価している
- peer_evaluation_guide を読んでいない

**対応:**
```
「なぜ+1だと思いますか？
 具体的に、どのような貢献がありましたか？」
```

---

### 5.4 システムの問題

#### **問題: イベントカードが足りない**

**対応:**
- 使用済みカードをシャッフルして再利用
- 閾値イベントは使い回さない

---

#### **問題: ステータスシートの計算が合わない**

**対応:**
- 各ステータスの変化を一つずつ確認
- イベントの status_effects と他者評価を分けて計算
- 必要に応じて、次のループで修正

---

#### **問題: 閾値イベントが連続発生**

**対応:**
- cooldown_loops（冷却期間）を確認
- 同じイベントは1ループ空ける
- 回復条件を満たすまで、他のイベントは引かない

---

## 6. デブリーフィング（振り返り）

### 6.1 デブリーフィングの目的

セッション終了後の振り返りは、**学習の定着**において最も重要なフェーズです。

**目的:**
- 体験を言語化する
- 他職種への理解を深める
- 現場への応用を考える

---

### 6.2 デブリーフィングの進行（30分）

#### **6.2.1 チーム評価の共有（5分）**

```
「お疲れさまでした。まず、チーム評価を見てみましょう。」

（team_evaluation_sheet を見せる）

「【最終評価】
- 患者中心性: 6.8
- 安全保障: 6.5
- ケアの持続可能性: 5.2
- 家族支援: 6.0

どの評価軸が高く、どの評価軸が低かったですか？
なぜそうなったと思いますか？」
```

---

#### **6.2.2 個人の振り返り（10分）**

```
「次に、個人で振り返りをしましょう。
 以下の質問に、1-2分で答えてください。」

【質問】
1. 今日の体験で、最も印象に残ったことは何ですか？
2. 他職種の視点で、新しく気づいたことはありますか？
3. 自分の職種の特徴や偏見に、気づいたことはありますか？
4. 現場で試してみたいことはありますか？
```

---

#### **6.2.3 グループでの共有（10分）**

```
「では、今の振り返りを、グループで共有しましょう。
 全員が1回ずつ話してください。」

【GMの役割】
- 全員が話せるように促す
- 深い気づきがあれば、「もう少し詳しく聞かせてください」
- 他の参加者にも「〇〇さんの話を聞いて、どう思いましたか?」
```

---

#### **6.2.4 全体まとめ（5分）**

```
「今日の体験を、現場でどう活かせるでしょうか？

【GMからのメッセージ】
- 価値観の対立は『悪』ではありません。
  多様性の表れであり、より良いケアのための材料です。
- 他職種の視点を理解することで、
  連携の質が高まります。
- 今日の気づきを、ぜひ現場で試してみてください。

ありがとうございました。」
```

---

### 6.3 デブリーフィングで避けるべきこと

- **「正解」を提示する**: 多様な答えを尊重する
- **特定の職種を批判する**: 個人攻撃にならないよう注意
- **現実の職場の問題を持ち込む**: ゲーム内の体験に焦点を当てる
- **長時間の説教**: GMの話は最小限に

## 7. よくある質問（FAQ）

### 7.1 セッション準備について

**Q: 初めてGMをします。何を準備すればいいですか？**
A: 「2.1 必要な物品」と「2.2 事前準備のチェックリスト」を参照してください。特に、役職カードとイベントカードの印刷を忘れずに。

**Q: 複数グループを同時に見るのは難しくないですか？**
A: 本システムはプレイヤー主導なので、GMの負担は軽いです。各グループを巡回し、質問に答えるだけで大丈夫です。

---

### 7.2 進行について

**Q: プレイヤーが「正解」を求めてきます。どう答えればいいですか？**
A: 「正解はありません。それぞれの職種の視点では、それぞれが正しいんです。大事なのは、お互いの視点を理解することです。」

**Q: 時間が足りなくて、Phase が終わりません。**
A: 「時間管理も学びの一部です。現場でも時間は限られています。ここで区切りましょう。」と伝えて、次のPhaseに進んでください。

**Q: 議論が停滞しています。介入すべきですか？**
A: まず、様子を見ましょう。1-2分待っても動かなければ、「〇〇についてはどう考えますか？」とヒントを出してください。

---

### 7.3 他者評価について

**Q: 評価が甘すぎる/厳しすぎる気がします。介入すべきですか？**
A: ループ1では様子を見ましょう。ループ2で「評価基準を確認しましょう」と促してください。

**Q: 評価の根拠が不明確です。どうすればいいですか？**
A: 「なぜそう思いますか？」と質問してください。根拠を言語化することも学びの一部です。

**Q: プレイヤーが「GMが決めてください」と言ってきます。**
A: 「このゲームでは、皆さん自身で評価します。GMは決めません。」と明確に伝えてください。

---

### 7.4 トラブル対応について

**Q: 議論が感情的になっています。どうすればいいですか？**
A: 「5分間、休憩しましょう」と宣言してください。休憩後、「今は理解し合う時間です」と再確認してください。

**Q: 患者・家族役が萎縮しています。**
A: 「患者さん、今の話をどう感じましたか？」と直接質問してください。専門職に対しても「患者さんの気持ちを確認しましょう」と促してください。

**Q: Xカードが使われました。どうすればいいですか？**
A: すぐに議論を止め、「休憩しましょう」と宣言してください。理由は聞かず、話題を変えてください。

## 8. GMのスキルアップ

### 8.1 初心者GMへのアドバイス

1. **完璧を目指さない**: 最初は時間管理だけに集中しましょう
2. **プレイヤーを信頼する**: プレイヤーは自分たちで進められます
3. **観察に徹する**: 介入は最小限に
4. **記録は後回し**: 進行を優先し、記録は後で補完してもOK

---

### 8.2 経験を積んだGMへのアドバイス

1. **微妙な介入**: 議論が浅いときに、さりげなく深掘りの質問を
2. **グループ間の調整**: 複数グループの進行速度を揃える
3. **デブリーフィングの深化**: より深い気づきを引き出す質問を
4. **記録の精度向上**: measurement_log を詳細に記録

---

### 8.3 GMコミュニティ

- 他のGMと情報交換する
- セッション後に振り返りをする
- 困ったことを共有する
- 成功事例を共有する
