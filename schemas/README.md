## システム構造（5つのJSONスキーマ）確立済み

### 1. scenario_template.schema.json
- シナリオの基本構造定義
- profession_cards_configに`objectives`追加済み（職種別目標）
- `difficulty_hint`追加済み（easy/normal/hard統一）

### 2. constraint_set.schema.json
- 時間/経済/リソース/制度の4軸制約
- flexibility（交渉可能性）実装
- gm_effect_guide（即座の影響と議論トリガー）

### 3. event_card.schema.json
- generation_support追加（AI生成支援）
- 現場別・重症度別バリエーション対応
- 16枚の在宅COPDイベント作成済み

### 4. dispute_queue.schema.json
- Phase2→Phase3への争点引き継ぎ
- conflict_positions（各職種の立場）記録
- dispute_番号での管理体系

### 5. measurement_log.schema.json
- 質的評価を主軸に転換済み
- profession_goal_achievement（職種別目標達成度）追加
- 数値指標は運営確認用に格下げ

## 核心設計思想（変更禁止）

### 基本コンセプト
- **価値観対立の体験→理解促進**（解消ではなく理解）
- **マーダーミステリー式時間管理**（30分×3ループ強制進行）
- **質的体験重視**（数値は運営確認用）
- **現実的制約の重視**（理想論回避）

### Phase構造（30分×3ループ固定）
1. **情報収集（5分）** ※ループ2以降スキップ
2. **争点露出（7分）** 
3. **価値観表明（7分）**
4. **方針決定（6分）**
5. **イベント・結果確認（5分）**

### 8タグ体系（確定・変更禁止）
`safety / time / resources / family_burden / patient_preference / function / legal_policy / coordination`

### 方針決定の優先順位
1. Safety（重大リスク回避）
2. Legal/Policy（法令・制度順守）
3. Time（緊急度）
4. Resources（現実的資源）
5. Patient Preference（本人希望）
6. Family Burden（家族負担）
7. Function/QOL（機能・生活）
8. Coordination（連携効率）

## AI活用方針

### シナリオ生成方法（確立済み）
1. 5つのJSONスキーマをAIに学習
2. 実際の事例情報を入力
3. AIが完全なシナリオセット生成
4. 人間が最終調整・品質確認

### 推奨AIモデル
- **メイン生成**: Claude Opus 4（日本語自然性・医療現場理解）
- **論理チェック**: GPT o3-pro（スキーマ準拠・整合性）
- **量産用**: Claude Sonnet 4（バランス・効率）

## 実証計画
- **江戸川実証**: 月3-5回、3-5人可変
- **測定重点**: 質的体験（相互理解・自己相対化・対立耐性・現場転用意欲）
- **フォロー**: 1週間後・3ヶ月後調査

## 重要な注意事項
1. **時間設計変更禁止**（30分×3ループ固定）
2. **8タグ体系変更禁止**
3. **価値観対立は解消しない**（理解促進に留める）
4. **職種目標は「チーム方針の内容」で評価**（発言回数ではない）
5. **争点記録はdispute_番号で管理**
