# 多職種連携ゲーム開発プロジェクト - 継続セッション

## プロジェクト概要
私は訪問看護師兼ゲームクリエイターとして、多職種連携研修用TRPGを開発しています。
現在、プロトタイプv1.0が完成し、システムの基本設計が確立されています。

### **使用AIモデル**

- **主開発:** Claude Sonnet 4 (日本語・現場感覚・質的評価)
- **構造検証:** GPT-5 (論理整合性・JSON検証)
- **最終検証:** o3-pro (深い検証・エッジケース・予定)

### **ファイル管理**
- **フォルダ構成:** 確立済み（上記参照）
- **命名規則:** scenario_id, constraint_set_id, event_id 統一

### 完成済み成果物
- **プロトタイプv1.0「複雑な家族関係とケアマネ信頼再構築」**
  - 73歳男性COPD患者、在宅酸素拒否、複雑な家族関係
  - 完全実装済み、即座に実証可能

## 確立済み設計思想（変更禁止）
### **核心コンセプト（変更禁止）**

- **「価値観対立の体験→理解促進」** (解消ではなく理解)
- **「質的体験重視」** (数値は運営確認用に格下げ)
- **マーダーミステリー式時間管理** (研修適合のため強制進行)
- **現実的制約の重視** (理想論回避)

### **評価システム（重要な方針転換済み）**

**主要評価（質的）:**

- 相互理解の深化（他職種への新しい気づき）
- 自己職種の相対化（自分の職種の特徴・偏見への気づき）
- 対立への耐性向上（価値観対立に対する心理的負担軽減）
- 現場転用意欲（学んだことの現場適用への具体的意欲）

**補助評価（運営確認用）:**

- 発言分布の偏り（安全確認）
- 時間配分適切性（進行改善）
- 争点多様性（コンテンツ品質）
- ルール遵守度（システム安定性）

### **Phase構造（30分×3ループ・固定）**

1. **情報収集(5分)** → 2. **争点露出(7分)** → 3. **価値観表明(7分)** → 4. **方針決定(6分)** → 5. **イベント・結果確認(5分)**
- ループ2以降はPhase1スキップ
- 強制時間進行でリアルな時間制約を再現

## 現在の開発方針

### シナリオ生成方法
**AI支援半自動化**を採用（プログラミング不要）
- 5つのJSONスキーマをAIに学習させる
- 実際の事例情報を入力
- AIが完全なシナリオセットを生成
- 人間が最終調整・品質確認

### 推奨AIモデル
- **メイン生成**: Claude Opus 4（日本語自然性・医療現場理解）
- **論理チェック**: GPT o3-pro（スキーマ準拠・整合性）
- **量産用**: Claude Sonnet 4（バランス・効率）
